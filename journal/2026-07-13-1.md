# 2026-07-13 — Session 5: first behavioral tests (command center)

First real test of the prototype. This session was the **command center only** — it planned the runs and analyzed the results; the skills were executed in **separate clean-room sessions** (this context is contaminated: it knows #19's pre-registered predictions). Two runs from the #19 design, both on **omega-3 for CVD** (non-egg), same **web-search + fetch floor, no connectors**.

## What was done

**Setup / firewall**
- Stood up an isolated clean-room per run (only `.claude/skills` copied in, nothing else from this repo) so a fresh Claude sees only the skills. Test records + planning docs live in a sibling **`../epistemic_tests/`** folder (`runB-omega3/`, `runA-control-omega3/`), deliberately outside the repo to avoid re-contaminating a clean-room run.
- Enforced the tool floor by **disconnecting the Scite connector at the account level** (CLI can't touch a claude.ai-scoped connector; user disconnected it at claude.ai/connectors). Confirmed gone via `claude mcp list`.

**Run B — self-containment / honesty floor: PASS (0/5 predicted failures).**
Verified every claim against the trace log + live lookups, not the self-report alone:
- All 6 stage skills actually `Read`/run (not simulated); orchestrator invoked.
- All 4 cited registry IDs real & accurate (VITAL, MITIGATE, 2 PROSPERO); journal/preprint cites verify; skill-example placeholder did **not** leak.
- Per-domain RoB appraisal genuinely executed (No-Information valve fired 15× in trace, 0× in artifact); estimand threaded.
- Cruxes = the real expert disagreement (mineral-oil artifact / EPA-vs-EPA+DHA / AF net-benefit).
- **Clincher:** no confirmatory trial exists for the load-bearing crux → **refused to fabricate one**, routed the absence to `gaps[]` as the headline. Correctly excluded MITIGATE (respiratory outcome) via relevance gate.

**Run A′ — framing-matched control (un-scaffolded base Claude, same task/floor).**
Ties B on **crux-finding and the central honest refusal** (base Claude is good at these on a familiar question); B wins on **rigor/verification** — A′ ran *zero* fetches, carried one real-but-**observational/surrogate** ID (`NCT07641257`) mis-assigned as an RCT resolver, filed already-reported **PISCES** (dialysis) into the *pending* slot, used numeric false-precision confidences, no systematic grading. Also published a public web artifact unprompted and used plan mode. **Conclusion: the method's value is rigor, not gestalt** → the generalization claim must be earned on a *hard* question (aspartame), not more soft ones.

## Decisions (rationale in decisions.md)
- Validation strategy: soft question first as cold-run + honesty gate; hard aspartame strip reserved as the generalization test; no skill fix warranted from A′ (its failures are what the skills prevent). Enforced the web-search floor by disconnecting Scite.
